# 🌊 Tide

**Cravings come in like waves. Tide helps you ride them out.**

Tide is a pocket companion for the single moment that actually decides the outcome — the few minutes when a craving peaks. Instead of a tracker you fill in after the fact, it's one calm button you reach for *when the urge hits*. It runs a guided ~10‑minute session to help you ride the wave out; tracking, insights, and streaks fill in automatically from those moments, so logging is never a chore.

Works for **overeating / junk food, alcohol, and smoking or vaping** with one shared engine.

👉 **Live app: https://aoprisan.github.io/tide/**

It's a Progressive Web App — install it to your home screen and it works offline.

## The "ride the wave" session

1. **Name it** — what you're pulled toward, and how strong (1–10). That's the whole ask up front.
2. **Breathe + surf** — a full‑screen wave rises and falls with a breathing pacer while a timer counts the craving down. Cravings crest and pass in 3–15 minutes on their own; the animation shows it literally.
3. **Do something** — optional quick actions (the 4 Ds: water, a short walk, text an anchor, brush teeth…). Tide learns which ones work for you.
4. **Remember why** — your own reasons, money saved, and how far you've come.
5. **Close the loop** — how it went and what set it off. No shame either way; a slip is logged neutrally as data.

The grounding techniques: **urge surfing**, **the 4 Ds**, and the **CBT cue→craving→routine→reward loop**.

## Tech

Plain static files — HTML, CSS, and vanilla JS, no dependencies or build step. All data is stored locally in your browser (`localStorage`); nothing is sent anywhere. PWA via a service worker + web app manifest.

```
index.html            app shell (all screens)
css/styles.css        visual system
js/app.js             state, the flow, computed insights, anchor, PWA wiring
manifest.webmanifest  installability
sw.js                 offline cache
icons/                app icons (generated by tools/make_icons.py)
```

### Run locally

The service worker and manifest need a real origin, so serve the folder rather than opening the file:

```bash
python3 -m http.server
# then open http://localhost:8000/
```

### Deploy

Pushing to `main` builds and publishes to GitHub Pages via `.github/workflows/deploy.yml`.

## Privacy

Tide keeps everything on your device. There are no accounts, no analytics, no servers. Clearing your browser data clears your history.
