# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repo is

Tide is a working **installable PWA** built from the original concept. It started as a design doc plus a clickable prototype; that has since been implemented as a real, offline-capable app with persistent local data and computed insights. There is no build step or package manager — it's plain static files (HTML + CSS + vanilla JS).

Files:

- `index.html` — the real app shell (all screens as `.view` sections + the `#deep` wave canvas).
- `css/styles.css` — the full visual system; mobile-first, fills the screen on phones and becomes a framed card on wide screens.
- `js/app.js` — all logic: state (localStorage), onboarding, the "ride the wave" flow, computed insights, the editable anchor, the living-wave canvas engine (`Waves`), hash routing, install prompt, and service-worker registration.
- `fonts/` — self-hosted variable woff2 (Fraunces roman + italic, Hanken Grotesk) so type works offline. Re-download with `tools/` notes below if ever needed.
- `manifest.webmanifest`, `sw.js`, `icons/`, `favicon.svg` — PWA plumbing (offline cache, installability, icons). Icons are generated by `tools/make_icons.py` (PIL).
- `.github/workflows/deploy.yml` — deploys the repo root to GitHub Pages on every push to `main`. Live at `https://aoprisan.github.io/tide/`.
- `Tide_Design_Doc.md` — the product design document. Still the source of truth for *what* Tide is and *why* every screen exists.
- `Tide_Prototype.html` — the original static concept mockup, kept for reference. **Not** the app — `index.html` is.

## Running / previewing

It's a static site, but the service worker and manifest need a real origin, so serve it rather than opening the file directly: `python3 -m http.server` from the repo root, then visit `http://localhost:8000/`. (`open index.html` mostly works too, but `file://` disables the service worker and install prompt.)

After changing any cached asset, bump the `CACHE` constant in `sw.js` so clients pick up the new version. After changing the icon design, re-run `python3 tools/make_icons.py`.

### Data model

All user data lives in `localStorage` under `tide.state.v1` (see `DEFAULT_STATE` in `app.js`): selected habits, reasons, the note-from-a-strong-day, anchor people, if-then plans, and the `sessions` array. **Insights, stats, streaks, and savings are all computed from `sessions`** — logging is a side effect of riding a wave, never a separate chore. The onboarding "Load a sample" link seeds plausible demo data via `seedAndExplore()`.

## The product, in one idea

Tide is built *backwards from the moment a craving peaks*. It's the thing a user reaches for when an urge hits (for overeating, alcohol, or smoking/vaping) — one calm button that runs a guided ~10-minute session to ride the urge out. Tracking, insights, and streaks are secondary and are fed automatically by those sessions, so logging is never a chore. The same engine serves all three habits; only the triggers, replacement actions, reasons, and savings math differ per habit.

The grounding behavioral techniques (worth preserving in any change): **urge surfing** (cravings crest and fall in 3–15 min on their own — the wave animation visualizes this literally), **the 4 Ds** (Delay/Distract/Deep-breathe/Drink — these are the in-session quick actions), and the **CBT cue→craving→routine→reward loop** (logging trigger/location/emotion surfaces personal cues).

## App structure (index.html)

Screens are `.view` sections, one shown at a time via the `active` class (`go(id)` switches them):
- `#v-onboard` — first-run setup (pick habits → name → first reason), or "Load a sample". Skipped once `state.onboarded` is true.
- `#v-home` — the dominant urge button + quiet stats (waves ridden, calm streak, money saved), all computed live.
- `#v-insights` — patterns computed from `state.sessions` (win rate, day×time heatmap, triggers, what-helps, savings). Gentle empty state until data exists.
- `#v-anchor` — the user's reasons, note-from-a-strong-day, anchor people, if-then plans — all editable (add/remove inline), filterable by habit.
- `#v-flow` — the signature **"ride the wave"** session, an overlay of sequential `.step` sub-screens: `s1` name it → `s1b` intensity → `s2` breathe/wave → `s3` quick actions → `s4` anchor → `s5` close the loop → `s5b` reflect (trigger chips + re-rate) → `sdone`. Finishing appends a session to `state.sessions`.

(The original two-phone `Tide_Prototype.html` mockup shares the same screen ids and CSS idiom, so changes often map across both — but `index.html` is the real app.)

Key JS functions (`js/app.js`): `go(id)` switches top-level views and re-renders the target; `renderHome`/`renderInsights`/`renderAnchor` rebuild a screen from `state`; `startFlow()`/`resetFlow()` open and reset the session; `next(from,to)` advances steps; `startWave()`/`stopWave()` drive the breathing pacer, countdown timer, and rotating reassurance copy; `setOutcome(kind)` then `logSession()` write the session and `showDone(kind)` branches the closing copy by outcome (`good`/`mid`/`slip`). Stats are pure functions over `state.sessions` (`wavesRidden`, `winRate`, `calmStreak`, `money`, `tallyFor`); the insights heatmap is computed from real session timestamps (day-of-week × 3-hour bucket).

## Design constraints to honor

When editing the prototype or building on the design, these are load-bearing, not decoration:

- **Tone is never shaming.** A slip is logged neutrally as data ("Okay. That happened. It doesn't erase anything."), never as a failed streak. See §7 of the design doc for in/out-of-voice examples.
- **One thumb, ten seconds, zero shame.** The urge button is reachable with no login wall; the up-front ask in a session is only "name it + rate intensity," everything else is skippable.
- **Visual system** (design doc §9, extended): the app's direction is *"bioluminescent deep"* — abyssal navy (`--abyss #04101e`) → phosphor teal (`--teal #2DD4BF` / `--phosphor #8cfbed`), with warm **sand** (`--sand`) reserved for the emotional "reasons/anchor" moments (a deliberate warm/cold temperature shift — keep it). Slow tidal easing (`--ease`, 550–1000ms); **the wave is the brand** — it's a live `<canvas id="deep">` (the `Waves` engine in `app.js`) that breathes behind every screen and *swells in time with the breathing pacer* during a session. Type is **Fraunces** (serif, emotional/display) + **Hanken Grotesk** (UI), self-hosted. All tokens live in the CSS `:root`.
- **Out of scope for v1** (design doc §8): calorie databases, barcode scanners, meeting finders, gamified avatars, social networks. Don't add these.
